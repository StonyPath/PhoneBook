این پروژه یک Phone Book Web API است که با    NET 10. ساخته شده و عملیات کامل CRUD را برای مخاطبان و فیلتر کردن بر اساس برچسب‌های دلخواه پشتیبانی می‌کند.

سیستم به چهار لایه تقسیم شده که هر کدام یک مسئولیت واحد دارند. لایه‌ی دامنه کاملاً با C# نوشته شده و قوانین دنیای واقعی را از طریق value objects اعمال می‌کند – شماره تلفن‌ها به ارقام نرمال‌سازی می‌شوند، نام‌ها برای خلاصه سازی امکان پذیرفتن داده از نوع string را می پذیرد ولی امکان تبدیل آن ها نیز به valueObject می باشد، و برچسب‌ها به صورت شناسه‌های الفبایی-عددی با حروف کوچک ذخیره می‌شوند. موجودیت PhoneBookRecord تمام منطق تغییرات را درون خود نگه می‌دارد.

موارد استفاده مانند «افزودن مخاطب» یا «ویرایش مخاطب» به صورت command query و  با MediatR در لایه‌ی Application پیاده‌سازی شده‌اند. اعتبارسنجی ورودی در یک pipline MediatR با استفاده از FluentValidation انجام می‌شود، بنابراین درخواست‌های نامعتبر هرگز به لایه‌ی دامنه نمی‌رسند. 
قوانین  bussiness که نیاز به داده دارند – مانند بررسی تکراری نبودن شماره تلفن – در handlerها قرار گرفته‌اند و در صورت نقض، exception سفارشی دامنه (NotFoundException, ConflictException) را ایجاد می‌کنند. 
لایه‌ی infrastructure، رکوردها را از طریق پیاده‌سازی یک repository در حافظه داخلی  ذخیره می‌کند، اما به‌راحتی امکان جایگزینی با Entity Framework در آینده را فراهم می‌کند.

لایه‌ی نمایش API یک لایه‌ی از کنترلرهای ASP.NET Core به همراه مستندات Swagger است. یک ExceptionHandler، تمام Exception دامنه را دریافت کرده و آن‌ها را به کدهای وضعیت HTTP مناسب (400، 404، 409) نگاشت می‌کند.

کل پروژه با مجموعه‌ای جامع از تست‌های واحد با استفاده از xUnit و Moq پوشش داده شده است که قوانین دامنه، قوانین اعتبارسنجی، handlerهای دستورات و خود repository را پوشش می‌دهند.
